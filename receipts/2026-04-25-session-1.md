## What was changed

### Doctrine update
Standing rule on session lifecycle was upgraded from v1.0 to v1.1.

- **v1.0** (2026-04-25 earlier): agent must end a turn with `Session disposable — safe to kill. Reason: <why>.` when no further attention needed.
- **v1.1** (2026-04-25 same day): two-step protocol. Before signaling disposable, agent must scan the session for carry-forward value (architecture decisions, doctrine changes, infra completions, surprising findings, deprecations, project state, user feedback) and write to memory hub. Then declare `Sedimented: <memo IDs>` or `Sedimented: nothing — routine session, no carry-forward value`. THEN emit the disposable signal.

Rationale: disposable-without-sediment leaks the session's learning. The two-step plugs the leak. Bar is auditable — the user can spot-check whether the agent applied the bar honestly.

### Tool exploration
Computer Use (mac-use-mcp) was deployed to navigate Omi's web dashboard for a historical data export. Two notable failures:
1. Mis-click on Omi sidebar (target was bottom-of-sidebar profile menu; landed on Discord invite link above it). Page navigated to `discord.com/invite/...`.
2. Mis-click on `/settings` mobile-splash overlay (target was "Continue to web anyway" link; landed on the App Store CTA above it because hit-region extended further than visible bounds).

Both failures: 100% from coordinate guessing in a non-Accessibility-exposed React app (UI tree returned only Chrome chrome, not Omi DOM). Lesson logged: when AX tree is empty, prefer URL-based navigation over coordinate clicks; resize window to trigger the desktop layout instead of fighting the mobile splash.
